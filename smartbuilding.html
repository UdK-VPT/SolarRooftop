Smart Building

Das funktionale Zusammenspiel der verschiedenen Technologien im Rooftop-Gebäude wurde über die Open Source-Plattform openHAB realisiert. Auf dieser Grundlage können unterschiedliche Szenarien für eine regelbasierte Gebäudeautomation definiert, im Experiment untersucht und flexibel angepasst werden.

openHAB unterstützt die parallele Einbindung verschiedener Bussysteme (z.B. im Rooftop-Gebäude den Modbus und den seriellen Bus) und Geräte über sogenannte „Bindings“. Hierdurch werden alle technischen Komponenten auf einer abstrakten Ebene als softwaretechnische Komponenten repräsentiert. Über algorithmisch definierte Regeln zwischen diesen Komponenten werden die übergeordneten Funktionen für das Smart Building festgelegt.

Anwendungsbeispiel: der im Innenraum gemessene CO2-Gehalt überschreitet auf Grund einer hohen Raumbelegung einen kritischen Grenzwert. Infolgedessen wechselt die Farbe der Innenraumbeleuchtung von weiss auf rot und gibt so den Gebäudenutzern den Hinweis das Lüftungsverhalten anzupassen.

Smart Mirror als Interface zur Nutzerkommunikation (Bildquelle: UdK Berlin)

Die Nutzerkommunikation wird im Rooftop-Gebäude über einen Smart Mirror unterstützt. Dieses Interface zeigt verschiedene aktuelle Informationen zum Gebäudezustand (z.B. Ladezustand der Hausbatterie), zur Gebäudeumgebung (z.B. Wetterinformationen, Abfahrzeiten des ÖPNV) oder zur Gebäudenutzung (z.B. Belegungsplan) an.
